<objective>
Turn the existing Phase 4 UI engineering rules into an executable style and AI-generation contract. This plan adds the financial semantic token layer, page archetype primitives, archive/dossier generation rules, and Claude skill guardrails needed before bulk page rewrites.
</objective>

<threat_model>
No external security boundary is introduced. Main risks are design-system drift and accidental regression from overly broad UI abstractions.

- Risk: Generated UI bypasses semantic tokens and reintroduces hardcoded Tailwind colors.
  - Mitigation: Add explicit forbidden patterns to skill/checklist and verify with grep/lint.
- Risk: New components duplicate existing primitives instead of composing them.
  - Mitigation: Read existing `Surface`, card, table, stats, and route components before creating new primitives.
- Risk: Financial colors become decorative and pollute Claude-like calm surface.
  - Mitigation: Limit `--exposure-*` usage to charts, legends, holdings, and evidence tables.
</threat_model>
