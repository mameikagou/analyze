<objective>
Refactor the remaining high-value frontend pages to follow the Phase 4.5 Style Contract using the primitives already created in 04.5-01. This plan is intentionally written for Kimi or another coding agent to execute page-by-page without taking over GSD project management.
</objective>

<scope>
## In Scope

This plan is only about single-page visual/layout refactors:

1. Refactor `/funds` using the Archive List Page archetype.
2. Refactor `/funds/$code` using the Fund Dossier Page archetype.
3. Refactor `/` using the Research Dashboard Page archetype.
4. Convert ad-hoc page layouts to the established primitives:
   - `PageShell`
   - `PageHeader`
   - `Surface`
   - `ArchiveTable`
   - `MetricValue`
   - `SignalBadge`
5. Remove forbidden hardcoded visual patterns:
   - `bg-white`
   - `text-gray-*`
   - `text-red-*`
   - `text-green-*`
   - `shadow-lg`
   - `shadow-xl`
6. Keep each task scoped to one page at a time.

## Out of Scope

Do not assign these to Kimi in this plan:

- Do not decide whether Phase 4, Phase 4.5, or v1.0 is complete.
- Do not update `ROADMAP.md`, `STATE.md`, `REQUIREMENTS.md`, milestone audit status, or traceability tables.
- Do not redesign product direction.
- Do not add backend APIs.
- Do not implement `/api/chat`.
- Do not fix SQL/security debt from the milestone audit.
- Do not batch rewrite all pages in one pass.
- Do not create another design system.
- Do not center work around `.claude/skills/claude-ui-system`; Kimi should treat the Style Contract and actual primitives as source of truth.
</scope>

<threat_model>
No external security boundary is introduced. The main risks are UI regressions, accidental data-flow changes, and design-system drift.

- Risk: Kimi rewrites API hooks or routing while refactoring UI.
  - Mitigation: Tasks explicitly forbid data-flow, route, and backend changes.
- Risk: Kimi preserves admin-dashboard structure for compatibility.
  - Mitigation: Plan requires replacing conflicting ad-hoc layout with Phase 4.5 archetypes.
- Risk: Kimi introduces forbidden Tailwind classes.
  - Mitigation: Every task has grep-based acceptance criteria for forbidden classes.
- Risk: Kimi over-expands scope into multiple pages.
  - Mitigation: Each task modifies exactly one route page.
</threat_model>
