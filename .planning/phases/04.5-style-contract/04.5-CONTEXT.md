# Phase 4.5: Style Contract + AI Generation Contract - Context

**Gathered:** 2026-04-26
**Status:** Ready for planning
**Source:** Direct planning edit from guide session

<domain>
## Phase Boundary

Phase 4.5 不改变产品方向，不重做 Phase 0/4，不批量重写页面。它只做一件事：把 Phase 4 已经完成的 Token、组件、暗色模式、动画、移动端基础，升级成 AI 能稳定执行的页面生成契约。

本阶段服务后续页面大改：`/funds/$code`、`/screening`、`/funds`、`/`、`/chat`、`/backtest`。但本阶段本身只要求建立契约、补 token、修 planning 漏洞和验证一个示例页面计划，不要求一次性翻新所有页面。

## Actual Situation

当前已有基础：
- `web/src/styles/tokens.primitive.css` 已有 Stone 暖灰、Orange accent、Red/Green/Blue 功能色、字号、间距、圆角、动画 token。
- `web/src/styles/tokens.semantic.css` 已有 `--bg-*`、`--text-*`、`--border-*`、`--accent-*`、shadow、overlay、scrollbar 等 semantic token。
- `web/src/styles/tokens.component.css` 已有 Composer、Message、Code Block、Artifact Panel、Button、Input、Card、Tooltip、Toast 组件级 token。
- `.planning/phases/04-ui-renovation/04-UI-SPEC.md` 已定义 Claude-like 基础风格：Stone、Surface、克制动画、暗色模式、移动端、accessibility。
- `web/src/design/博主列表.html` 有有价值的 Archive Finance 原型：dense table、mono 数字、弱阴影、扫描线、档案馆气质。
    
当前真实缺口：
- 不是缺 Token 基础，而是缺 Composition Contract。
- 不是缺组件，而是缺 Page Archetype。
- 不是所有建议都值得马上做；最有落地价值的是能约束 AI 后续生成页面的规则。

</domain>

<decisions>
## Implementation Decisions

### D1 — Must Land: Composition Contract
必须落地。当前 UI-SPEC 能防硬编码，但不能决定页面结构。新增 `04.5-STYLE-CONTRACT.md`，明确页面只能由 PageShell、PageHeader、Surface、ArchiveTable、DossierLayout、Metric/Signal primitives 组合。

### D2 — Must Land: Page Archetypes
必须落地。只保留 4 个强落地 archetype：
1. Research Dashboard Page — `/`
2. Archive List Page — `/funds`, `/screening`
3. Fund Dossier Page — `/funds/$code`
4. Chat / Composer Page — `/chat`

Backtest Lab Page 暂不作为 Phase 4.5 核心，因为 Phase 4 已经翻新过 `/backtest`，后续只作为已存在页面的检查对象，不再优先设计母版。

### D3 — Must Land: Financial Semantic Tokens
必须落地，但缩小范围。新增 signal / exposure / score / risk 四类 semantic token。不要一次性设计几十种行业色；先定义 8 个 exposure token 足够覆盖持仓/行业暴露图。

### D4 — Must Land: Archive Finance Variant
必须落地，但不能照搬 `博主列表.html`。该原型的 Slate/Clay/Moss/Steel 不能直接进入正式系统。正式方向是 Claude Stone base + Archive Finance variant：全局仍是 Stone，Archive 只定义数据密度、表格行高、mono 数字、弱行 hover、少量证据色。

### D5 — Must Land: Forbidden Patterns
必须落地。现有禁令不够。必须继续禁止 `shadow-lg`、`shadow-xl`、`border-gray-*`、`text-red-*`、`text-green-*`、随意渐变、裸 Tailwind card。所有页面容器必须走 `Surface` 或页面 archetype 组件。

### D6 — Must Land: Fix Planning Contradiction
必须落地。`04-02-PLAN.md` 示例中直接使用 `var(--green-500)` / `var(--red-500)` 违反 Primitive 禁用规则。必须在 Style Contract 中声明历史 plan 示例不得作为新代码 token 使用范式；后续示例统一用 `--signal-positive` / `--signal-negative`。

### D7 — Downgrade: Source Serif
降级处理。`Source Serif 4` 有档案馆味道，但引入新字体会增加复杂度，也可能破坏 Claude-like 稳定性。Phase 4.5 不引入 serif 字体；只保留 Inter + mono。若要 serif，只能用于后续品牌实验，不进入核心契约。

### D8 — Downgrade: Scan-line Animation
降级处理。`博主列表.html` 的扫描线动画有记忆点，但不是核心。Phase 4.5 只保留 “row enter stagger / subtle hover / active background” 规则，不强制实现扫描线。

### D9 — Defer: Blogger Collision Product Layer
暂缓。博主碰瓷是引流层，但不是当前 Style Contract 的落地点。本阶段只把 Archive List archetype 设计成可复用，后续能承载博主列表，不在本阶段新增业务页面。

### D10 — Reject: Full Bloomberg Terminal Direction
明确拒绝。不要把产品做成 Bloomberg / Wind 终端。目标是 Claude 写出来的基金研究档案馆，不是高密度金融终端。

</decisions>
