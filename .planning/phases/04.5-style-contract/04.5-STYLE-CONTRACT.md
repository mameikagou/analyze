# Phase 4.5: Style Contract + AI Generation Contract

**Phase:** 04.5-style-contract
**Date:** 2026-04-26
**Purpose:** 把现有 Claude-like UI 工程规范升级成 AI 可执行的页面生成协议。

---

## 1. Non-Negotiable Direction

本项目的前端目标不是 shadcn admin，不是 Bloomberg terminal，也不是普通数据后台。

目标是：

> Claude 写出来的基金研究档案馆。

具体含义：

- Claude base：Stone 暖灰、低对比、轻边框、少阴影、大留白、克制动画。
- Archive finance variant：数据区可以更密，数字用 mono，表格像档案柜，颜色只服务证据和状态。
- 不追求炫技，不追求金融终端压迫感，不追求 SaaS 营销大渐变。

---

## 2. What Actually Has Landing Value

### Must Land

1. **Composition Contract** — 决定页面结构，防止 AI 自由拼后台。
2. **Page Archetypes** — 给首页、列表、详情、聊天固定母版。
3. **Financial Semantic Tokens** — 金融状态和暴露颜色必须有语义。
4. **Archive Finance Variant** — 吸收 `博主列表.html` 的密度和档案馆气质。
5. **Forbidden Patterns** — 防止后续 AI 写烂 Tailwind。
6. **Visual Acceptance Checklist** — 让“高级感”可以验收。

### Downgrade / Defer

1. **Source Serif** — 暂不进入核心系统，避免字体复杂度和 Claude base 冲突。
2. **Scan-line Animation** — 暂不强制，只保留 subtle row motion。
3. **Backtest Lab Archetype** — `/backtest` 已翻新，本阶段不优先。
4. **Blogger Collision Page** — 是后续引流页面，不是当前 style foundation。

## 3. Composition Contract

任何新页面必须先选一个 archetype，不允许从空白 `div` 开始自由拼。

### Universal Page Rules

每个页面最多包含：

- 1 个 PageHeader
- 1 个主叙事区
- 2-4 个关键指标区
- 1 个主数据区
- 0-1 个辅助解释区
- 0-1 个操作区

信息顺序必须是：

```text
结论 / 状态 → 关键指标 → 证据数据 → 可执行动作
```

禁止：

```text
先堆表格 → 再堆卡片 → 最后补标题
```

## 4. Page Archetypes

### 4.1 Research Dashboard Page

用于：`/`

```text
PageShell
  PageHeader
    eyebrow: FUND SCREENER
    title: 全市场趋势筛选
    description: 一句话��释今天市场筛选状态
    actions: primary refresh / secondary view screening
  InsightStrip
    3-4 MetricCard
  MainSurface
    TopCandidatesList or ArchiveTable preview
  SecondaryGrid
    DataQualityCard
    MarketStatusCard
    RecentBacktestCard
```

Rules:

- 首页不是营销 landing，也不是后台 dashboard。
- 首屏必须说明“今天有什么值得看”。
- MetricCard 不得超过 4 个。
- 主视觉焦点只能是 Top candidates 或 market summary 二选一。

### 4.2 Archive List Page

用于：`/funds`, `/screening`, future blogger archive

```text
PageShell
  PageHeader
    eyebrow
    title
    description
    status summary
  FilterBar
    tabs
    compact controls
    sort indicator
  ArchiveSurface
    ArchiveTable
      mono code column
      tabular numeric columns
      subtle hover
      right-edge action affordance
  Pagination or LoadMore
```

Rules:

- 表格必须包在 ArchiveSurface 内，不能直接铺在页面 canvas 上。
- 行高默认 52-56px。
- 代码、百分比、金额、排名必须使用 mono + tabular nums。
- 表头 12px、regular/medium，不要 bold 大标题感。
- hover 只能用 `--bg-hover`，不能加阴影。
- 涨跌/强弱只能用 signal token，不能用 `text-red-*` / `text-green-*`。

### 4.3 Fund Dossier Page

用于：`/funds/$code`

```text
DossierPage
  DossierHeader
    identity: code + name + market
    diagnosis summary
    action group
  DiagnosisGrid
    3-4 SignalCard
  EvidenceLayout
    left: chart / exposure / timeline
    right: holdings / explanation / risk notes
  TimelineSurface optional
```

Rules:

- 这是产品核心页，不允许 AI 自由发挥。
- 页面必须像“研究档案”，不是“详情表单”。
- 首屏必须给出诊断摘要，而不是只展示基本信息。
- 持仓、行业、净值走势属于 evidence，不是 decoration。
- 行业色只能出现在 exposure chart、holding tags、legend，不得染页面背景。

### 4.4 Chat / Composer Page

用于：`/chat`

```text
ChatShell
  ConversationSurface
  ArtifactPanel optional
  Composer
```

Rules:

- 这是最接近 `claude.ai/new` 的页面。
- Composer 必须使用已有 composer tokens。
- AI 输出里的金融解释应复用 Dossier / Archive 的展示组件，不要在 chat 内另造一套卡片。

## 5. Surface Hierarchy

最多三层 surface：

```text
canvas → surface → elevated
```

Allowed:

```tsx
<Surface variant="surface" bordered rounded="xl">
  ...
</Surface>
```

Allowed only for popover/dropdown/modal:

```tsx
<Surface variant="elevated" shadow rounded="xl">
  ...
</Surface>
```

Forbidden:

```tsx
<div className="bg-white rounded-xl shadow-lg p-6">
```

Nested surfaces:

- 允许最多 2 层嵌套。
- 第 3 层只能是 table row / badge / input，不得再是 card。
- 不允许卡片套卡片套卡片。

## 6. Financial Semantic Tokens

Add to `tokens.semantic.css`.

```css
:root {
  --signal-positive: var(--green-600);
  --signal-negative: var(--red-600);
  --signal-neutral: var(--stone-500);
  --signal-warning: var(--orange-600);

  --signal-positive-subtle: var(--green-50);
  --signal-negative-subtle: var(--red-50);
  --signal-neutral-subtle: var(--stone-100);
  --signal-warning-subtle: var(--orange-50);

  --exposure-tech: #3b82f6;
  --exposure-consumer: #f97316;
  --exposure-financial: #64748b;
  --exposure-healthcare: #22c55e;
  --exposure-industrial: #a855f7;
  --exposure-energy: #eab308;
  --exposure-cash: var(--stone-400);
  --exposure-other: var(--stone-500);

  --score-strong: var(--signal-positive);
  --score-medium: var(--signal-warning);
  --score-weak: var(--signal-negative);

  --risk-low: var(--signal-positive);
  --risk-medium: var(--signal-warning);
  --risk-high: var(--signal-negative);
}

.dark {
  --signal-positive-subtle: rgba(34, 197, 94, 0.14);
  --signal-negative-subtle: rgba(239, 68, 68, 0.14);
  --signal-neutral-subtle: var(--stone-800);
  --signal-warning-subtle: rgba(249, 115, 22, 0.14);
}
```

Exception:

- `--exposure-*` may use raw hex inside token file because token files are the boundary.
- Components must never use raw hex or primitive color tokens directly.

## 7. Archive Finance Variant

Archive variant is not a new theme. It is a layout and density mode for evidence-heavy areas.

Use it for:

- fund list
- screening table
- holdings table
- blogger archive later
- backtest trade history table

Archive rules:

```text
row height: 52-56px
header font-size: 12px
cell font-size: 14px
numeric font: mono + tabular nums
border: 1px solid --border-subtle
hover: --bg-hover
shadow: none or token-only micro shadow
radius: 6px-12px depending container
```

Do not copy from `博主列表.html`:

- Slate palette
- Clay/Moss/Steel primitive names
- Source Serif dependency
- direct CSS utility classes
- JS table rendering style

Do keep:

- dense-but-calm rows
- mono numeric rhythm
- archive/table wrapper
- right-edge hover affordance
- restrained evidence colors

## 8. Typography Rules

Base:

- Page title: 24px / semibold / tight
- Section title: 18px / semibold
- Body: 15px
- Secondary: 14px
- Label: 11-12px / uppercase optional / muted
- Numeric: mono + tabular nums

Forbidden:

- giant marketing headings on app pages
- random serif emphasis
- bold table headers
- multiple font families beyond sans + mono

## 9. Forbidden Patterns

Hard ban in generated UI:

```text
bg-white
bg-black
bg-gray-*
bg-slate-*
text-gray-*
text-slate-*
text-red-*
text-green-*
border-gray-*
border-slate-*
shadow-lg
shadow-xl
from-*
to-*
via-*
```

Conditional ban:

```text
rounded-full
p-8
p-10
gap-10
gap-12
```

Allowed only when explicitly justified by component contract.

Structural ban:

- No naked card divs.
- No page-level arbitrary gradients.
- No status color as decoration.
- No table outside ArchiveSurface.
- No primitive token in component code.

## 10. Component Usage Matrix

| Need | Must Use | Must Not Use |
|------|----------|--------------|
| Page container | PageShell | raw full-page div with colors |
| Page heading | PageHeader | ad-hoc h1 block |
| Card/panel | Surface | `div.bg-white.rounded...` |
| KPI number | MetricCard / MetricValue | raw `span.text-green-*` |
| Signal state | SignalBadge | `text-red-*`, `text-green-*` |
| Data table | ArchiveTable | raw table directly in page |
| Fund detail | DossierLayout | generic dashboard grid |
| Chat input | Composer | custom textarea card |

If a component does not exist yet, the executor should create it instead of bypassing the contract.

## 11. AI Generation Checklist

Before generating or modifying a page, AI must answer internally:

1. Which archetype is this page?
2. What is the single primary focus of the first screen?
3. Which data is conclusion, which is evidence, which is action?
4. Which components from the usage matrix are required?
5. Are all containers Surface/PageShell-derived?
6. Are all status colors semantic financial tokens?
7. Does any class violate forbidden patterns?
8. Does the result look like Claude research UI, not admin dashboard?

---

## 12. Visual Acceptance Checklist

A page passes only if:

- First screen has one clear visual focus.
- PageHeader explains what the page is for.
- Surface hierarchy does not exceed 3 levels.
- Color is used only for status, exposure, link, or focus.
- Numeric data uses mono/tabular rhythm.
- Tables match Archive density.
- Dark mode is low-glare, not simple inversion.
- No naked Tailwind cards exist.
- No hardcoded Tailwind color classes exist.
- Screenshot does not resemble shadcn admin dashboard.

```tsx

```

These examples are not valid going forward.

```tsx
var(--signal-positive)
var(--signal-negative)
```

Any new or modified component must use semantic or financial semantic token names only.
