# Fund Screener 当前进度（说人话版）

> 更新日期: 2026-04-18
> 面向对象: 任何人，不需要懂量化，也不需要懂 Python

---

## 一句话说清这个项目是干嘛的

这是一个**帮你挑基金的工具**。

你跑一条命令，它会自动去各大金融数据网站（东财、Tushare、Yahoo Finance 等）抓取 A 股基金、美股 ETF、港股 ETF 的净值数据，然后算出一批**趋势在涨**的基金，最后生成一份 Markdown 报告。你可以直接把报告丢给 Claude/Gemini 让 AI 帮你进一步分析。

它还有一个**本地数据库**，把你每次跑的数据都存下来，时间越长，你拥有的历史数据就越多，能做的事也越复杂。

## 已经做到哪一步了（按版本讲）

### v0.1 ✅ 基础版：能跑起来了（已完成）

最朴素的版本，核心就干一件事：
- 从三个市场（A 股 / 美股 / 港股）抓基金数据
- 算 MA20（20 日均线）和 MA60（60 日均线）
- 只保留 **MA20 > MA60** 的基金（简单说就是"最近涨得还行"的）
- 生成 Markdown 报告

这个阶段就像是你雇了一个实习生，每天帮你扫一遍全市场，告诉你哪些基金现在趋势向上。

### v0.2 ✅ 数据湖 + 分析工具（已完成）

以前数据抓完就丢了，现在**全部存进本地 SQLite 数据库**。

好处是：
- 今天抓的数据明天还能用
- 可以回头问问题，比如："过去一周哪些基金趋势最强？"
- 基金持仓变化、行业分布、风格漂移，这些需要"对比今天和之前数据"的事，现在能做了

**新增了三把分析"手术刀"：**
1. **动量扫描**: 找出"均线多头 + 今天刚好回调"的基金（适合右侧买入的潜力标的）
2. **风格漂移检测**: 看基金经理有没有偷偷换赛道（本来说是买科技股的，结果偷偷买白酒了）
3. **持仓相关性矩阵**: 避免你买了两只看起来名字不同、其实底层持仓高度重合的基金

还加了一个**异步批量抓取**，可以一次性拉几百只基金的数据。

### v0.3 ✅ 申购限额标注（已完成）

A 股基金有个很烦人的问题：**有些基金不让买，或者每天只能买一点点**。

这个功能就是解决这个：
- 自动标注每只基金的申购状态（正常买 / 限额买 / 暂停买 / 状态未知）
- 报告里按这四类分组展示
- 默认只标注、不剔除，避免你错过好基金
- 你也可以手动开启过滤，把限额太低的基金筛掉

### v0.4（当前正在做）量化打分引擎

以前工具只会说"这只基金均线向上，算通过了"。现在它要变得更聪明：

**给每只基金算三个分数：**
- **动量分**: 最近涨得好不好
- **夏普分**: 涨得好，但风险（波动）大不大
- **回撤分**: 跌的时候跌得狠不狠

然后标准化、加权，算出一个**综合评分**，按分数从高到低排，只给你看 Top 30 或 Top 50。

> 简单理解：以前是个"开关"（通过/不通过），现在是个"排行榜"。

**另外刚完成一个架构升级：A 股数据源不再绑死一个提供商。**

以前 A 股数据只能从东财（akshare）抓，现在做成了"组合数据源"：
- 基金列表从 akshare 抓（最全）
- 净值历史从 tushare 抓（更规范、更稳定）
- 以后加第三个数据源，改改配置就行，不用改代码

2. **补完端到端验证**
   - 把进度文档里挂着的几个待验证项跑一遍，确认没 bug

3. **历史数据回填**

4. **回测模块**
   - 这是"研究系统"和"玩具"的分水岭。能回测才算真量化

5. **定时任务 / Web UI**
