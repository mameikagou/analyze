# Checklist

## AI Generation Checklist

1. Which archetype is this page?
2. What is the single primary focus of the first screen?
3. Which data is conclusion, which is evidence, which is action?
4. Which components from the usage matrix are required?
5. Are all containers Surface/PageShell-derived?
6. Are all status colors semantic financial tokens?
7. Does any class violate forbidden patterns?
8. Does the result look like Claude research UI, not admin dashboard?

## Visual Acceptance Checklist

1. First screen has one clear visual focus.
2. PageHeader explains what the page is for.
3. Surface hierarchy does not exceed 3 levels.
4. Color is used only for status, exposure, link, or focus.
5. Numeric data uses mono/tabular rhythm.
6. Tables match Archive density.
7. Dark mode is low-glare, not simple inversion.
8. No naked Tailwind cards exist.
9. No hardcoded Tailwind color classes exist.
10. Screenshot does not resemble shadcn admin dashboard.
